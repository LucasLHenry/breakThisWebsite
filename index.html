Everyone has heard of unsafe websites, but what does it actually mean?

If a website is "unsafe", that tipically means it is vulnerable to at least one of these types of attacks.

SQLi

(structured query language injection)

Manipulation of items in a database using a coding language called SQL. This abuses the vulnerability of making the computer think that your search is not just text, but actual instructions. This primarily affects the server and not the users, but it can be made to also affect users.

CSRF

(cross site request forgery)

Tricking a user into submitting a malicious request to a website that requires credentials by using those already obtained by logging in to the website. This can be done by making a link that does the malicious action and tricking the victim into clicking on it.

XSS

(cross site scripting)

Injecting malicious javascript through user input into an average website to be executed as part of the it. It is usually inputed through normal input fields, and can sometimes corrupt the entire website. This is only doable if the website does not implement proper input sanitization.

Why does this matter to you?

The fact is that bad web security effects everyone, and the best way to protect yourself is to learn how it works

If you use an online store vulnerable to SQLi, any attacker could steal your username, password, credit card number, and everything you've every purchased with only one line of code.

If you have an account at a bank, and their website is vulnerable to CSRF, your entire balance could be solen and your account deleted just by clicking a link.

If you're on a blog that's vulnerable to XSS and someone happens to have written a comment that uses XSS, you could be redirected to a fake website that tricks you into giving up valuable infromation without you even knowing anything has happened.

How can you protect yourself?

There are lots of things to look for on an unsafe website, here are some of them:

General

  • Look for https instead of http in the url bar. That means that all the information you send to the website is encrypted (scrambled in a special way) so that no one else can understand it.
  • Look for the lock 🔒 symbol in the address bar. That means that the website is using the proper security standards. Note: this does not guarrantee that a website is safe. There have been examples of malicious websites that still get the lock icon.
